# Pointer Consistency

Top-level governance files each own a distinct responsibility.

- `PROGRESS.md`: milestone, done, in-progress, pending
- `NEXT_STEP.md`: one directly executable next action
- `MEMORY.md`: durable lessons, stable boundaries, recurring pitfalls
- `AGENT_INDEX.md`: route by task type and active skill taxonomy

## Checks

- `NEXT_STEP.md` must contain one actionable pointer, not a backlog.
- `PROGRESS.md` in-progress work must match the same active loop named by `NEXT_STEP.md`.
- `MEMORY.md` should not become a second progress log.
- `AGENT_INDEX.md` must not route to retired skills.
