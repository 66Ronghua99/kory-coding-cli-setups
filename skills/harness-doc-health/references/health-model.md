# Health Model

Repository truth is checked in four layers:

1. bootstrap integrity
2. pointer consistency
3. explicit `spec -> plan -> evidence` relationships
4. freshness of execution and verification truth

`AGENTS.md` should act like a concise map into that truth, not a long manual that duplicates it.

The principle is simple: if top-level pointers and active documents disagree, the repository is already drifting even before code is inspected.
