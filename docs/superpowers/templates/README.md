# Template Index

## Required Templates

- `SPEC_TEMPLATE.md`: use for approved design intent
- `PLAN_TEMPLATE.md`: use for executable implementation plans
- `CHANGE_REQUEST_TEMPLATE.md`: use for changes requested after scope is already set
- `EVIDENCE_TEMPLATE.md`: use for validation and delivery evidence

## Usage Rules

- New behavior or workflow changes must start with `SPEC_TEMPLATE.md`.
- Implementation plans must reference an approved spec.
- Scope changes during execution must be classified before implementation changes.
- Completion claims should point to an evidence document or equivalent artifact path.
