# NEXT_STEP Reference Template (Strict)

用于定义“当前唯一下一步动作”，保证会话中断后可直接续跑。

## 必填规则

- 全文必须是单段（建议 1 句）。
- 必须包含：阶段名 + 执行动作 + 验收证据 + 输出物。
- 必须与 `PROGRESS.md` 的 `P0-NEXT` 语义一致。

## 禁止项

- 禁止多条 backlog。
- 禁止记录历史过程。
- 禁止仅写目标不写验收证据。

## 初始化骨架（可直接复制）

```markdown
# NEXT STEP

进入“<阶段名称>”阶段：<动词开头的执行动作>，并以 <可观察证据/验收标准> 作为完成标记，输出 <工件/日志/文档路径>。
```
